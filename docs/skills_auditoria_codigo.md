# Skills para auditoría, corrección de código y buenas prácticas

Este documento agrupa las skills del catálogo que resultan más útiles para revisar código, detectar problemas, refactorizar y aplicar estándares de calidad en proyectos de software.[cite:1]

## Auditoría de código

Estas skills sirven para inspeccionar una codebase, detectar problemas estructurales y encontrar causas raíz en errores o decisiones técnicas débiles.[cite:1]

- `production-code-audit`
- `error-detective`
- `analyze-project`
- `architect-review`

## Corrección y refactor

Estas skills están orientadas a limpiar código, reorganizar estructuras y modernizar proyectos sin limitarse a corregir errores puntuales.[cite:1]

- `code-refactoring-refactor-clean`
- `codebase-cleanup-refactor-clean`
- `framework-migration-legacy-modernize`

## Buenas prácticas y arquitectura

Este grupo se enfoca en clean code, principios SOLID, manejo correcto de errores, seguridad técnica y diseño de arquitectura mantenible.[cite:1]

- `uncle-bob-craft`
- `software-architecture`
- `error-handling-patterns`
- `memory-safety-patterns`
- `cloudformation-best-practices`

## Testing y calidad continua

Estas skills ayudan a validar cambios, prevenir regresiones y sostener buenas prácticas mediante pruebas y automatización de calidad.[cite:1]

- `testing-patterns`
- `e2e-testing-patterns`
- `workflow-patterns`
- `github-actions-templates`

## Grupo recomendado

Para un stack web o full-stack, el conjunto más equilibrado para auditoría, corrección y calidad técnica es el siguiente.[cite:1]

- `production-code-audit`
- `code-refactoring-refactor-clean`
- `uncle-bob-craft`
- `error-detective`
- `error-handling-patterns`
- `testing-patterns`
- `e2e-testing-patterns`
- `software-architecture`
